# 📦 Permission Admin 구조 정리 (Spring Boot)

## 1. 🧩 Permission (Entity)

```java
@Entity
@Table(name = "permissions")
public class Permission
```

### 역할

* DB의 `permissions` 테이블과 매핑되는 JPA 엔티티
* 하나의 권한 정보를 표현

### 필드

* `id` → PK (자동 증가)
* `name` → 권한 코드 (예: `MENU_READ`, `USER_DELETE`)
* `description` → 권한 설명

### 특징

* `@NoArgsConstructor(PROTECTED)` → JPA 기본 생성용
* setter 없음 → 불변성 유지 구조

---

## 2. 📤 PermissionResponse (DTO)

```java
public class PermissionResponse
```

### 역할

* API 응답용 데이터 객체 (Entity 직접 노출 방지)

### 포함 데이터

* `id`
* `name`
* `description`

### 특징

* `@AllArgsConstructor`
* 단순 데이터 전달용 (읽기 전용 DTO)

---

## 3. 📁 PermissionRepository

```java
public interface PermissionRepository extends JpaRepository<Permission, Long>
```

### 역할

* DB 접근 계층 (DAO 역할)
* Spring Data JPA가 구현 자동 생성

### 기능

* 기본 CRUD 자동 제공

  * `findAll()`
  * `findById()`
  * `save()`
  * `delete()`

---

## 4. ⚙️ PermissionAdminService

```java
@Service
public class PermissionAdminService
```

### 역할

* 비즈니스 로직 처리 계층
* Repository → DTO 변환 담당

### 핵심 메서드

```java
public List<PermissionResponse> getPermissions()
```

### 흐름

1. `permissionRepository.findAll()` → DB 조회
2. Stream 처리
3. Entity → DTO 변환
4. List 반환

### 특징

* `@RequiredArgsConstructor` → DI 자동 처리
* Controller와 DB 사이 중간 계층 역할

---

## 🧭 전체 흐름

```
Controller
   ↓
Service (PermissionAdminService)
   ↓
Repository (PermissionRepository)
   ↓
DB (permissions table)
```

```
Entity → DTO 변환
Permission → PermissionResponse
```

---

## ✅ 핵심 요약

* **Entity** → DB 구조
* **DTO** → API 응답 구조
* **Repository** → DB 접근
* **Service** → 비즈니스 로직 + 변환 담당
