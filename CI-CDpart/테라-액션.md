# ⚙️ 2. GitHub Actions 설명서 (CI/CD DOC)

> 목적: “코드가 어떻게 빌드/배포되는가”

---

## 🧱 1. 문서 역할

| 항목 | 내용                        |
| -- | ------------------------- |
| 목적 | CI/CD 자동화                 |
| 대상 | Dev / Backend / Frontend  |
| 범위 | build, test, push, deploy |
| 제외 | 인프라 생성 (Terraform만 호출)    |

---

## 🔄 2. 전체 파이프라인

```text
Code Push
   ↓
Build
   ↓
Test
   ↓
Docker Build
   ↓
Registry Push (GHCR/ECR)
   ↓
Terraform Apply
   ↓
Deploy
```

---

## 🧪 3. CI 단계

| 단계       | 설명            |
| -------- | ------------- |
| checkout | 코드 가져오기       |
| setup    | Node / JDK 설치 |
| test     | unit test     |
| lint     | 코드 품질 검사      |

---

## 🐳 4. Docker 단계

```yaml
- name: Build Image
  run: docker build -t myapp .
```

---

## 📦 5. Registry Push

| 환경   | 저장소  |
| ---- | ---- |
| dev  | GHCR |
| prod | ECR  |

---

### GHCR 예시

```bash
docker tag myapp ghcr.io/org/myapp:latest
docker push ghcr.io/org/myapp:latest
```

---

### ECR 예시

```bash
aws ecr get-login-password | docker login
docker tag myapp <ecr-url>:latest
docker push <ecr-url>:latest
```

---

## 🚀 6. Deploy 단계 (Terraform 호출)

```yaml
- name: Terraform Apply
  run: |
    cd terraform/envs/dev
    terraform init
    terraform apply -auto-approve
```

---

## 🔐 7. 환경 분리 전략

| 브랜치     | 환경      |
| ------- | ------- |
| dev     | dev     |
| main    | prod    |
| release | staging |

---

## ⚙️ 8. CI/CD 핵심 원칙

* CI와 Infra 완전 분리
* Terraform은 “배포만”
* GitHub Actions는 “흐름만”
* Image는 immutable (tag 기반)

---

## 📊 9. 역할 분리 표

| 영역             | 책임                |
| -------------- | ----------------- |
| GitHub Actions | 빌드 + 테스트 + 배포 트리거 |
| Terraform      | 인프라 생성            |
| Docker         | 실행 단위             |
| Registry       | 이미지 저장            |
